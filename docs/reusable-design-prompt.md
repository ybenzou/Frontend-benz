# 可复用前端设计 Prompt

请将项目设计为专业、高密度、长期使用的工作台，而不是营销页或常见 AI Dashboard 模板。

## 技术与架构

- 使用 Next.js、React、TypeScript 和 Tailwind CSS，不使用原始 HTML/CSS 作为项目架构。
- 组件、布局、数据访问和业务逻辑分层；UI 原语不包含业务数据。
- 保留后端 API、Cloudflare 部署和运行时数据校验的接入空间。

## 字体与信息层级

- 正文使用清晰的专业无衬线字体，数字使用等宽字体和 tabular numerals。
- 优先使用本地静态字重，避免 Windows 下变量字体模糊或合成字重。
- 通过字号、字重、明度和对齐建立主次；关键数字明显大于标签和辅助说明。
- 使用 `rem` 和响应式根字号，避免大屏只增加空白。

## 去卡片化

- 默认使用透明开放区域，不添加完整矩形外框、阴影或大圆角。
- 主要依靠排版、留白、背景明度和网格对齐建立内容分组。
- 完整边界仅用于核心数据面、独立交互区域、选中状态和浮层。
- 表格移除外围卡片；仅保留表头基准线、极弱行线和 hover 状态。

## 分割系统

- 使用统一的分段式基准线：细线、间隙、短强调线、间隙、细线。
- 分割线与标题、数字列或网格锚点对齐，不作为随机装饰。
- 一级数据使用强调背景或明显基准线；二级模块使用短线和留白；三级数据依靠排版与弱行线。
- 禁止霓虹、高亮发光、玻璃拟态和赛博朋克 HUD。

## 流式布局

- 使用 Grid、Flex、`minmax()`、`auto-fit`、`auto-fill` 和 intrinsic sizing。
- 填充区域使用 `flex: 1` 和 `min-width: 0`，避免逐断点手写固定宽度。
- 宽表格在自身容器滚动，不允许页面产生非预期横向溢出。
- 在 1920、1440、977 和 390px 下验证布局、裁切和空间利用率。

## 颜色与动效

- 使用低饱和深色中性背景；颜色只表达状态、交互和数据语义。
- 上涨、下跌、警告和强调色必须拥有固定语义，避免装饰性配色。
- 动效用于价格更新、状态变化和页面反馈，保持短促、克制且可中断。
- 支持 `prefers-reduced-motion`，不得牺牲可读性换取动感。

## 可复用视觉原语

- 建立 `OpenSection`、`SectionHeader`、`SegmentedRule`、`DataGroup`、`EmphasisSurface` 和 `TableRegion`。
- 统一管理颜色、间距、分割线、表面和字号令牌，禁止逐页面重复微调。

## 验收

- 信息密度接近专业研究终端，而不是宽松营销页面。
- 关键数据明显高于次级信息，移除边框后仍可快速判断分组。
- 无文字裁切、非预期横向溢出和无意义空白。
- TypeScript、Lint、测试和生产构建通过。
